Lesson 2

Creating a Structured Document with HTML:

Developer Tools:

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Tree-like structure of HTML:

A branching structure flow. The browser builds a tree-like structure of the page (using DOM), which basically turns HTML tags into elements that form a tree. Each element has an opening and closing tag which can then can be text or another element. This can be visualized as a structure of nested virtual boxes. Smaller elements/objects can be inside a box, which can be inside another box and so on. We define boxes with div tags. Using this branching structure along with tags and classes allows a programmer to avoid repeating code. For example using HTML WITH CSS a developer can apply a particular set of styles by just placing the HTML under the correct tag and class instead of typing all the style parameters over and over for each case.

How contents are structured in HTML:

DOM: Document Object Model. A standard convention for representing and interacting with elements of HTML. Each HTML tag creates an element in the DOM that the browser uses to display the page.
